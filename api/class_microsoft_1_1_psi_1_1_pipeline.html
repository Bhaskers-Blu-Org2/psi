EventHandler< PipelineExceptionNotHandledEventArgs > PipelineExceptionNotHandled
 Event that is raised when one or more unhandled exceptions occur in the pipeline. If a handler is attached to this event, any unhandled exceptions during pipeline execution will not be thrown, and will instead be handled by the attached handler. If no handler is attached, unhandled exceptions will be thrown within the execution context in which the exception occurred if the pipeline was run asynchronously via one of the RunAsync methods. This could cause the application to terminate abruptly. If the pipeline was run synchronously via one of the Run methods, an AggregateException will be thrown from the Run method (which may be caught). More...

Runs the pipeline synchronously in replay mode. This method may be used when replaying data from a store.

Parameters
replayIntervalThe time interval within which to replay the data. The pipeline will commence playback at the start time of this interval, and only messages bearing an originating time within this interval will be retrieved from the store(s) contained in the pipeline and delivered. ◆ PipelineExceptionNotHandled

EventHandler<PipelineExceptionNotHandledEventArgs> Microsoft.Psi.Pipeline.PipelineExceptionNotHandled

Event that is raised when one or more unhandled exceptions occur in the pipeline. If a handler is attached to this event, any unhandled exceptions during pipeline execution will not be thrown, and will instead be handled by the attached handler. If no handler is attached, unhandled exceptions will be thrown within the execution context in which the exception occurred if the pipeline was run asynchronously via one of the RunAsync methods. This could cause the application to terminate abruptly. If the pipeline was run synchronously via one of the Run methods, an AggregateException will be thrown from the Run method (which may be caught).
