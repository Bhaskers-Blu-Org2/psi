Detailed Description

Represents the configuration for the AudioSource component.

Use this class to configure a new instance of the AudioSource component. Refer to the properties in this class for more information on the various configuration options.

◆ AudioLevel

double Microsoft.Psi.Audio.AudioSourceConfiguration.AudioLevel

Gets or sets the audio input level.

This is the initial level to set the audio recording device to. Valid values range between 0.0 and 1.0 inclusive. If not specified, the current level of the selected recording device will be left unchanged.

◆ DeviceName

string Microsoft.Psi.Audio.AudioSourceConfiguration.DeviceName
getset

Gets or sets the name of the audio source device.

Use this to specify the name of the audio recording device from which to capture audio. To obtain a list of available recording devices on the system, use the AudioSource.GetAvailableDevices static method. If not specified, the default recording device will be selected.

◆ Gain

float Microsoft.Psi.Audio.AudioSourceConfiguration.Gain
getset

Gets or sets the additional gain to be applied to the captured audio.

This specifies an additional gain which may be applied computationally to the captured audio signal. Values greater than 1.0 boost the audio signal, while values in the range of 0.0 to 1.0 attenuate it. The default value is 1.0 (no additional gain).

◆ OptimizeForSpeech

bool Microsoft.Psi.Audio.AudioSourceConfiguration.OptimizeForSpeech
getset

Gets or sets a value indicating whether the captured audio should be pre-processed for speech recognition applications. Available in Windows 10 Creators Update or later.

In later versions of Windows 10, the audio capture pipeline optionally enables additional signal processing to apply enhancements such as echo and noise suppression to optimize the audio for speech recognition applications. By default, this option is set to false. This feature may not be available for all capture devices.

◆ OutputFormat

By default, audio will be captured in the default format of the audio recording device. Use this to specify a different format for the AudioBuffer Out stream of the AudioSource component.

◆ TargetLatencyInMs

int Microsoft.Psi.Audio.AudioSourceConfiguration.TargetLatencyInMs
getset

Gets or sets the target audio latency.

Captured audio will be output as a stream of type AudioBuffer in the AudioSource component. This parameter controls the amount of audio to capture for each AudioBuffer message, which in turn determines the latency of the audio (i.e. the amount of lag between when the audio was produced and when a captured AudioBuffer is output on the stream). The larger this value, the more audio data is carried in each AudioBuffer and the longer the audio latency. For live audio capture, we normally want this value to be small as possible, with the lower bound being constrained by the audio capture pipeline. By default, this value is set to 20 milliseconds. Is is safe to leave this unchanged.
