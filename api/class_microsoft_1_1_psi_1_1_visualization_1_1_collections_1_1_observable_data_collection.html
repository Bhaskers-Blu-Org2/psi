Detailed Description

The purpose of ObservableDataCollection is to optimize exposing non-bindable lists to XAML binding. It does this in two ways: 1) It surfaces as an ObservableCollection which minimizes item removals and resets. 2) It wraps non IPropertyChangeNotify objects so that they can be updated wholesale (via indirection in a ObservableDataItem) so that updates don't cause a collection change to force bindings to update.
